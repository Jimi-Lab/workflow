System Overview: VCCFinder builds a vulnerable commit database and learns a classifier over heterogeneous code/repository features to prioritize audit targets.
Pipeline Stages:
1. Select GitHub C/C++ projects and collect CVEs/fixing commits.
2. Use a blame heuristic to map fixing commits to VCCs.
3. Extract project/author/commit/file/function/text features.
4. Convert heterogeneous features into a generalized bag-of-words representation.
5. Train a linear SVM classifier.
6. Use classifier scores/explanations to prioritize commits for audit.
Core Algorithm: The paper's VCC heuristic ignores documentation, blames deleted lines, blames neighbors around inserted blocks, then marks the most-blamed commit(s) as vulnerable-contributing.
