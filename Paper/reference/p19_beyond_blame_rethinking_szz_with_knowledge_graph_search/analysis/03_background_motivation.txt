Domain Concepts Introduced:
- 2 Background and Related Work SZZ Algorithm and Its Variants.The SZZ algorithm [ 32] pioneered automated BIC identification by tracing deleted or modified lines in bug-fixing commits via git blame.
- Over two decades, numerous variants have improved precision through annotation graphs and cosmetic change filtering [19], meta-change exclusion [7], refactoring detection [24, 25], deep learning [35], and LLM- based reasoning [36, 40].
- Most recently, LLM4SZZ [36] leverages LLMs to rank blame candidates and expands context around modified lines for candidate collection.
- Despite these advances, all approaches share a fundamental limitation: they restrict the search space to commits returned by git blame, inherently missing BICs that require broader history traversal.
- AgenticSZZ addresses this limitation by structuring the commit history as a temporal knowledge graph and navigating it through agent-guided exploration.
- Knowledge Graphs (KGs) in Software Engineering (SE).A KG is a graph-based data structure where nodes represent entities and edges encode relationships between them, enabling queries that traverse and reason over these connections [16].
- For automated program repair, repository-aware KGs bridge semantic gaps between issue descriptions and code patches [ 45], and unified code KGs enable multilingual issue resolution [ 5].
- However, none of these works address BIC identification or model the temporal evolution of commits.
