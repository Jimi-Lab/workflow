Among baselines, traditional SZZ variants like B-SZZ achieve higher recall by returning multiple
candidates, but suffer from low precision; conversely, heuristics like R-SZZ improve precision but
sacrifice recall. AgenticSZZ achieves both the highest recall and highest precision by expanding
the search space through the TKG while using intelligent agent reasoning to filter candidates.
The recall improvements validate our RQ0 findings about BIC distribution.AgenticSZZ
achieves higher recall than LLM4SZZ across all datasets, with the largest gains on DS_GITHUB-j
(+0.174) and DS_LINUX (+0.091). These improvements reflect our RQ0 finding that 28% of BICs
require exploration beyond immediate blame commits. While LLM4SZZ’s context expansion can
reach some of these BICs, at first sight AgenticSZZ’s TKG-based history traversal seems to provide
a more systematic approach to exploring ancestor commits (a deeper analysis of this is performed
in RQ2). The simultaneous precision improvements indicate that the agent’s reasoning effectively
filters false positives introduced by search space expansion.
DS_APACHE presents unique challenges that limit improvement.While AgenticSZZ
matches LLM4SZZ on DS_APACHE, it does not achieve the same gains as on other datasets, and
the difference is not statistically significant (𝑝= 0.74). As shown in Figure 2, DS_APACHE has the
highest blame complexity: 30% of cases involve ≥10 deleted lines with ≥3 blame commits, whereas
DS_LINUX and DS_GITHUB concentrate on the low-complexity region (47% and 43% respectively
have 1 to 2 deleted lines with only 1 blame commit). This complexity increases the difficulty of
identifying the correct BIC among many candidates. Additionally, DS_APACHE projects span
diverse Apache Software Foundation repositories with varying coding conventions, making it
harder for the agent to reason about code relationships consistently.
