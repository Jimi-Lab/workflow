language processing can be used to lessen human efforts in the
partial matching. Errors in the developer logs and patches are
threats to validity. We have found some errors in the logs, al-
though the number of errors is very small in our evaluation, but it
will decrease the accuracy of the method if no additional checks
are introduced. Sometimes patches may not be implemented
correctly, which means that the software versions are vulnerable
even if they are patched. We plan to study how to identify
incorrect(incomplete or wrong) patches in the future work.
VIII. CONCLUSION
In this article, we proposed and implemented novel ideas
(by verifying CVE information with security patches and de-
veloper logs based on version tree) and techniques (including
partial matching and log tracing methods) to identify the known
vulnerable software versions with respect to the CVEs. By
comparison of the reports for Linux between our tool and the
NVD, we found that there were 11,827 incorrect vulnerable
versions of the NVD, and conﬁrmed that 94 patched versions
lacked committed records in the changelogs. For future research,
it will be interesting to solve the above limitations, and explore
whether partial matching and repatching can be extended for
binary code vulnerability detection.
