Limitations, Threats, and Ethics

Method limitations:
- Task mismatch for VulnVersion: KRYSTAL's evidence is runtime audit/provenance data, while VulnVersion must judge whether a software version includes vulnerable or fixed code.
- Query/rule dependence: benefits of SPARQL and declarative rules depend on high-quality ontology mappings and rule maintenance.
- Runtime cost: RDF lifting and SPARQL query execution introduce performance overhead relative to specialized custom graph data structures; the paper discusses this as a manageable tradeoff.
- Data dependence: DARPA-TC scenarios provide controlled red-team/blue-team data, but coverage of real enterprise logging variability may be limited. [NEEDS EVIDENCE] for broader deployment claims.

- Large sample logs and TTL outputs are present, but they were not validated against paper table values. [NEEDS TABLE REPAIR]

Ethics/security:
- The work supports attack discovery and forensic reconstruction; it also exposes how attack behavior can be represented and queried. Use in VulnVersion should emphasize defensive vulnerability triage and patch/affected-version validation.
