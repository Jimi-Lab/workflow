Experiment Design Analysis

Datasets:
- The paper evaluates on DARPA Transparent Computing datasets, organized into multiple categories/scenarios. The extracted text states the evaluated scenarios cover more than seven days of log data from three datasets and more than 53 GB of JSON log data.
- Local data artifacts under experiment/input include Cadets, Theia, Trace, and FiveDirections JSON samples. These are static evidence only; no scenario was replayed. [EXECUTION NOT REQUESTED]

Evaluation dimensions:
- Graph size reduction and compression: compares raw JSON log data with RDF provenance graph size and HDT compressed representation.
- Runtime performance: measures graph generation, tag propagation/query costs, and attack reconstruction time.
- Threat detection: compares tag-propagation-based alerting and signature/rule-based alerting against DARPA ground truth/high-level attack activities.
- Knowledge linking: demonstrates background linking and SPARQL federation as a qualitative capability.

Transferable experiment pattern:
- Evaluate each stage separately rather than only end-to-end. VulnVersion can similarly separate: evidence retrieval quality, patch-presence detection, version graph construction, target-version judgment, and final affected-version range metrics.
- Pair scale metrics with correctness metrics. For VulnVersion this suggests reporting repository size, number of versions/tags/branches inspected, commit traversal cost, and affected-version accuracy/recall.
- Include case studies that show graph traversal paths, not just aggregate metrics.
