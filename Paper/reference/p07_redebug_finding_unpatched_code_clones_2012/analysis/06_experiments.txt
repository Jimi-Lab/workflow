Research Questions:
The paper is not organized as explicit RQs, but the introduction frames three empirical questions: attacker discovery from patches, OS responsiveness to known vulnerabilities, and persistence of unpatched clones across OS versions.

Datasets:
The paper uses early/late 2011 source datasets including Debian Lenny, Ubuntu Maverick, Linux kernel, SourceForge C/C++ projects, Debian Squeeze, and Ubuntu Oneiric. Table I contains exact dataset sizes [NEEDS TABLE REPAIR].

Baselines:
Deckard is used for direct comparison in discussion/evaluation. Related work also discusses CCFinder, CP-Miner, Deja Vu, MOSS, SYDIT, and Pattern Insight Code Assurance.

Metrics:
- Build time.
- Query time.
- Database size.
- Number of unpatched code clones.
- Confirmed real bugs.
- False detection / missed clones in comparison with Deckard.
- Similarity distribution for copied code measurement.

Experimental Protocol:
The paper describes collecting source datasets twice in 2011, collecting Debian/Ubuntu security-related patches, applying existing package patches, then querying patch sets against source datasets under default n=4, c=3, theta=1 settings.

Implementation Details:
The paper states the evaluated system was about 1000 lines of C and 250 lines of Python and used QuickLZ compression. Local source is a separate Python reimplementation, not the original C implementation [NEEDS ARTIFACT].
