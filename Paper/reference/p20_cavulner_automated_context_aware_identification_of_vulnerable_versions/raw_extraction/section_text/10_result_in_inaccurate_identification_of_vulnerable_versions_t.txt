result in inaccurate identification of vulnerable versions. To address this issue, we performed a
comprehensive manual validation of all 176 CVEs and identified a list of vulnerable versions for
each.
Our annotation principle was to identify vulnerable code based on the fundamental nature and
semantics of the vulnerability. Specifically, for each CVE, we first analyze the CVE description and
its associated CWE information. Drawing on expert knowledge and publicly available vulnerability-
related resources, we identify the root cause of the vulnerability. We then examine the code before
and after the fix to determine the vulnerable code blocks. After that, we applied git blame to trace
these vulnerable lines through the version history. For each traced version, we examined whether
the previous version contained the vulnerability. If not, we consider the current version to be the
vulnerability-inducing version. If it did, we continued to trace the newly identified vulnerable
lines further back. This recursive process was repeated until we found a version in which the
vulnerability was no longer present. We define the range of vulnerable versions as the versions
that span from the first identified vulnerable version to the version where the vulnerability was
fixed. In total, this process resulted in a high-quality ground truth that includes 17,436 vulnerable
versions in 176 CVEs.
The entire manual analysis was conducted by a team consisting of one Ph.D. student with more
than ten years of experience in Java vulnerability research, including the discovery of zero-day CVEs,
and two master’s students. Each analyst performed in-depth research using personal experience and
publicly available resources, such as official advisories and technical blogs. We use Krippendorff’s
alpha [25] to measure the agreement among the three annotators. The Krippendorff’s alpha among
the three annotators is 0.78, indicating a substantial agreement. The three annotators resolved
the disagreements by group discussion until consensus was reached. The entire ground truth
construction process took about one month.
