Concrete Pain Point:

- The number of known software vulnerabilities has been increasing rapidly in recent years, which extensively influenced the ecosystem.
- To manage this growing threat, organizations such as the National Vulnerability Database (NVD) [ 31] and GitHub Advisory Database [ 2] curate critical information on disclosed vulnerabilities, to support the wide usage of software supply chain tools like Software Composition Analysis (SCA), to assess potential risks.
- In this case, SCA tools rely heavily on the accuracy of vulnerability data, especially the correct vulnerable versions, for automated mappings of security risks in downstream user projects.
- However, multiple studies [9, 16, 33, 42] have shown that these vulnerability records often contain inconsistencies or errors, which can seriously mislead maintenance decisions.
