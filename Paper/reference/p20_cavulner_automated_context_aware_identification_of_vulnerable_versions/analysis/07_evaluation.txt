Case Study:
- VERJava selected the TOP6 projects with respect to the number of CVEs having available patches.
- Additionally, it included Liferay-portal, which contains the highest number of CVEs in NVD.
- The CVEs in the VERJava dataset span a broad period of each project’s lifecycle, represent various vulnerability types, and involve patches of varying sizes.
- Our study specifically targeted CVEs from this collection that affect open-source projects hosted on GitHub.
- To construct our dataset, we merged the CVEs used by V-SZZ and VERJava, then excluded those whose vulnerable versions were not available on GitHub, such as tomcat 5.x.x or 6.x.x.
- As shown in Table 1, it resulted in a corpus of 176 CVEs across 43 distinct Java projects and encompassing all 8,730 versions, including popular projects such as jackson-databind [19], liferay- portal [29], jenkins [22], struts [7], tomcat [8], spring-framework [40] and spring-security [41].
